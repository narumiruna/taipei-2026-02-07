## 📁 ドキュメント構造

各都市ごとに以下の5つのファイルで情報を段階的に提供（Progressive Disclosure）：

```
gourmet/
  [都市名]/
    📌 overview.md    - 都市の概要・戦略・調査進捗（5分で読める）
    🏆 top-places.md  - 最終推薦リスト + Dining Strategy（予約はここから）
    📊 candidates.md  - 全候補店舗の一覧表（スコア・ステータス付き）
    📝 notes.md       - 詳細調査ノート・根拠資料（情報源リンク付き）
    🚫 excluded.md    - 除外店舗と除外理由（監査証跡）
```

### ファイルの使い分け
| ファイル | 用途 | 読む人 |
|---------|------|--------|
| overview.md | 都市の特徴と調査方針を素早く把握 | 初めて見る人 |
| **top-places.md** | **予約・訪問先の決定** | **旅行者**（最重要） |
| candidates.md | 候補全体を俯瞰、スコア比較 | プランナー、調査者 |
| notes.md | 根拠資料の確認、深掘り調査 | 調査の妥当性を検証したい人 |
| excluded.md | なぜ除外したかを確認 | 監査、再検討が必要な時 |

## 🔍 調査方法論

このプロジェクトは **エビデンスベースの調査手法** を採用：

- ✅ **複数ソースのクロスチェック**: Google Maps、食べログ、Michelin Guide等
- ✅ **50点満点の標準化スコア**: Taste/Quality, Value, Convenience, Consistency, Risk
- ✅ **トレーサビリティ**: すべての推薦に情報源リンクを記載
- ✅ **透明性**: 除外理由も含めすべて記録
